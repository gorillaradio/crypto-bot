# Frontend → shadcn/ui Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the crypto-bot dashboard frontend on a shadcn/ui + Tailwind v4 foundation, preserving the current look 1:1 (except the activity box) and fixing mobile overflow.

**Architecture:** Two ordered phases inside one big-bang delivery. Phase 1 stands up shadcn + Tailwind with its STOCK default theme and migrates every component onto shadcn primitives + Tailwind utilities, deleting the global stylesheet. Phase 2 swaps shadcn's stock colors for our existing OKLCH values so the palette matches today. The existing Vitest suite (jsdom, behavior-based) is the regression gate; browser verification at desktop + mobile widths is the visual/1:1 gate.

**Tech Stack:** React 19, Vite 8, TypeScript, Tailwind v4 (`@tailwindcss/vite`), shadcn/ui, recharts (unchanged), Vitest + Testing Library.

## Global Constraints

- Package manager is **npm** (`package-lock.json`). Use `npm` / `npx`, never `pnpm`.
- **Tailwind v4** (not v3): CSS-first config, no `tailwind.config.js`.
- **1:1 look:** no visual redesign of any screen. Only allowed visible change is mobile no longer overflowing horizontally.
- **EventsFeed (activity box) is OUT OF SCOPE.** Do not restyle it. Its global CSS classes must keep working. It is detailed later, separately.
- **No new features, no behavior changes, no backend changes.**
- Existing tests in `frontend/src/__tests__/*` must stay green. Update an assertion only when it checks a removed class/structure, and only to assert the new structure without weakening the behavior it verifies.
- Theming rule: reuse our OKLCH color *values*, mapped onto shadcn's standard variable names. Never bend the framework to keep old variable names (`--bg`, `--surface`, custom z-index scale). If a token can't map cleanly, the standard setup wins.
- All paths below are under `frontend/`. Run all `npm`/`npx` commands from `frontend/`.
